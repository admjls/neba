Highlights of this release

Introducing: Lazy loading

A key focus of NEBA has been - and always will be - performance. NEBA drives dynamic, high volume sites with vast amounts of content and strives to eliminate any overhead.

We are proud to make another addition to better performing websites through the introduction of lazy loading for resource models.

Starting with release 3.4.0, reference collections - such as @Children of a resource or lists of @References - are automatically and transparently lazy-loading. Furthermore, any mapped field can now be explicitly lazy-loading using the new Optional<T> interface.

Immediate performance gains

As one of the test beds for lazy loading, we used a complex AEM™ project featuring 100.000+ pages and high-volume traffic. An immediate performance gain resulted from upgrading to NEBA 3.4.0 - without a single line of code changed in the project itself:

NEBA 3.3.0 NEBA 3.4.0 Performance gain
Publishing environment
Average page load time (ms) 285 217 23%
Authoring environment
Average page load time (ms) 286 236 17%

Release 3.4.0 has no API changes - simply update to this NEBA version to benefit from the changes.
