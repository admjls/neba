Release 3.5.0

A new model statistics console for real-time visualization of resource model performance

In this release, the model metadata console was replaced with the model statistics console. This console visualizes resource model metadata collected in real-time and allows querying the collected data to find models with specific performance characteristics right in the browser. Try the interactive demo of the model statistics console.

First-class support for custom annotations

One of NEBA's key goals is to enable clean separation of the project-specific implementation and the system-level concerns of Sling, AEM™ or NEBA. Consequently, NEBA offers extension points in its API to allow adding additional functionality without depending on NEBA's implementation details.

The most common use cases for these extension points is supporting custom annotations. For instance, one project featured a @Translated annotation allowing the injection of internationalized values into resource models. Until now, such extensions could be implemented using custom post processors. Now, NEBA features first-class support for custom annotations. Not only does this enable easy addition of semantic annotations - it also allows extending NEBA's out-of-the-box annotations.

Support for meta-annotations

NEBA now supports meta-annotations (annotated annotations). This allows combining custom and NEBA annotations into new semantic annotations - for example

{% highlight java %} @Reference @Path("paths") @MyCustomAnnotation private List pages; {% endhighlight %}

Can now be simplified by annotating @MyCustomAnnotation with NEBA's annotations instead, like so:

{% highlight java %} @Reference @Path("paths") public @interface MyCustomAnnotation { ... } {% endhighlight %}

META-annotations are supported for all of NEBA's annotations, including MVC annotations.

Upgrade to Spring 3.2.13

This release updates from Spring 3.2.11 to Spring 3.2.13. No API changes are introduced by this change, however it is recommended to upgrade project dependencies to this Spring version as well. Please refer to the Spring 3.2.x release notes regarding the corresponding changes.

Upgrading from 3.4.0

Simply upgrade your dependencies and re-deploy NEBA - there are no incompatible API changes in this release. Note that the version of Spring contained in the delivery package was updated from 3.2.11 to 3.2.13, which should be reflected in the projects dependencies as well.
