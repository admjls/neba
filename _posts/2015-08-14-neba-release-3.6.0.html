Highlights of this Release

AEM 5.6 to AEM 6.1 support: Getting the filter order right

From AEM 5.6 to AEM 6.1, the meaning of the filter.order property of sling filters changed. To support both versions, NEBA 3.6.0 automatically detects the AEM version it is running on and adjusts the order of its filters accordingly. Using NEBA 3.6.x is thus strongly recommended for AEM versions > 5.6 or Sling versions with Sling engine > 2.3.2.

Handling rotated logfiles in the log file viewer

Since 3.6.0, the log viewer console automatically hides rotated logfile versions, giving administrators a focus on the most recent log files.

Improved filtering in the NEBA model statistics console

The model statistics console now got even better at automatically completing filter expressions. Any expression can now quickly be created by typing and applying the autocompletion suggestions.

HTTP OPTIONS and TRACE requests now passed to @Controllers

In Spring, OPTIONS and TRACE requests are usually answered with a default response by the DispatcherServlet. NEBA now overrides this default and routes these methods to registered controllers, thus enabling proper implementation of CORS for RESTful Spring controllers.

Upgrading from 3.5.0

Simply upgrade your dependencies and re-deploy NEBA - there are no incompatible API changes in this release. Note that the version of Spring contained in the delivery package was updated from 3.2.13 RELEASE 1 to 3.2.13 RELEASE 2, which should be reflected in project dependencies as well.
