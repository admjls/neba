Highlights of this release

Lazy-loading support for custom annotation mappers

NEBA explicitly supports lazy-loading fields via the Optional interface. With 3.8.0, fields with custom annotations can now use the same interface. The custom annotation processors will then be applied in a lazy fashion.

Release notes

Fixed bugs

#89 The neba:defineObjects may yield the wrong model if the same synthetic resource is mapped to two different models within the same request
#88 AnnotatedFieldMappers are not invoked on Optional<...> fields

Upgrading from 3.7.0

User of NEBA 3.7.0 can upgrade to 3.8.0 without any further changes.
