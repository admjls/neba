Highlights of this Release

Improved support of the Sling 8 standalone application

The Sling standalone application no longer ships with the bgservlets bundle by default. NEBA 3.10.0 thus made the dependency to the corresponding package optional. Consequently, Sling 8 users must no longer deploy the bgservlets bundle for NEBA to work (#108). Thanks to Mike Nimer for reporting this!

Improved method contract for @PostMapping and @PreMapping

With 3.10.0, methods annotated with @PostMapping or @PreMapping must no longer be public (#106), thus users are no longer forced to potentially publish implementation details as API. Thanks to djessup for the contribution!

Release notes

Improvements

#108 Make the dependency to bgservlets optional
#106 @PostConstruct only works on public methods, forcing developers to expose implementation contracts

Fixed bugs

This release contains no bug fixes.

Upgrading from 3.9.0

User of NEBA 3.9.0 can upgrade to 3.10.0 without any further changes.
