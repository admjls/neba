Highlights of this release

NEBA 4.0.0 ships with Spring 4

NEBA 4.0.0 ships with Spring 4, bringing with it improved Java 8 support and many more valuable features, such as support for WebSockets.

Exclusive Support for Java 8

Support for Java version less then 8 has been discontinued, as they have officially reached end of life and are no longer support by AEM 6.2. However, the NEBA 3.x branch - which is still actively maintained - continues to support these legacy versions.

Extended Spring MVC support

With NEBA 4.0.0, one can now leverage all features of Spring MVC - including those requiring a WebApplicationContext, for instance the Spring Forms Tag Library.

In addition, NEBA 4.0.0 also enables using Sling scripts as Spring Controller Views, thus fusing the power of Spring MVC views with Sling's mighty Script resolution.

Out of the box support for automated Spring MVC JSON responses

Automated Model Lookup for Adobe HTL (formerly known as Sightly)

You no longer need to couple your HTL (Sightly) view to an explicit model class - just use {% highlight xml %} data-sly-use.="/apps/neba/neba.js" {% endhighlight %} and NEBA will provide the most specific model automatically.

A brand-new, Web Sockets based Log Viewer

The NEBA log viewer now uses WebSockets to receive logfile updates in real time. In addition, advanced stacktrace detection was added to the viewer - and requests in the request.log are automatically linked to their response.

Support for AEM 6.1+ and Sling 8+

With AEM 6.1 and Sling 8, significant improvements have been introduced to the underlying frameworks, including breaking API changes. To better support these modern platforms, NEBA 4.x currently supports AEM versions 6.1+ and Sling 8+. Previous Platforms are supported by the 3.x branch of NEBA, which will be actively maintained until the EOL of AEM 6.0 core support in June 2017.

Adobe HTL (formerly Sightly) and Sling Healthchecks replace Velocity rendering and @SelfTest

Since early versions of AEM 5.x and Sling 7, features previously exclusive to NEBA have made it into the product core. To further the use of product best-practices, NEBA no longer ships with support for velocity as a template mechanism and with @SelfTest support. Users of these features can either continue to use NEBA 3.x, which is actively maintained until June 2017, or transition to Adobe HTL and Sling Health Checks.

Release notes

Improvements

    Fixed bugs

      Upgrading from 3.10.0

      NEBA 4.0.0 contains no breaking changes in the main Resource Model API, i.e. the annotations.

      NEBA 4.0.0 Supports AEM 6.1+. Users of previous AEM versions can continue to use NEBA 3.x, which is actively maintained until June 2017.

      NEBA 4.0.0 requires using JAVA 8+. Users of legacy Java versions can continue to use NEBA 3.x, which is actively maintained until June 2017.

      Users of @SelfTest are encouraged to transition to the platform standard, Sling Health Checks, as @SelfTest is discontinued in favor of these platform standards.

      Users of Velocity templating, for instance the <neba:render ...> JSP tag library, are encouraged to transition to either Adobe HTL or Thymeleaf, as Velocity support is discontinued in favor of these platform standards.

      Neba 4.0.0 ships with Spring 4.2.6. Project dependencies must be updated to the respective Spring bundle versions.
