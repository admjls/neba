Highlights of this release

NEBA 3.11.0 continues support for AEM 5.x - 6.1

With NEBA 3.x, we are continuously supporting users of older AEM / Sling versions, delivering on our promise to support these versions at least until the official EOL of AEM 6.1 in July 2017.

Release notes

Fixed bugs

    Upgrading from 3.10.0

    Users of NEBA 3.10.0 can update to this version without side effect.

    Updating your maven dependencies

    NEBA is built using Maven. Core, API and delivery packages containing NEBA and it's runtime dependencies are available from maven central. For developing with NEBA, including the API in your project is all you need to do:

    {% highlight xml %} io.neba io.neba.neba-api 3.11.0 {% endhighlight %}

    If you'd also like to browse or debug the core, add it to your project's dependencies like so:

    {% highlight xml %} io.neba io.neba.neba-core 3.11.0 {% endhighlight %}

    The convenience CRX deployment package for AEM can be referenced like so:

    {% highlight xml %} io.neba io.neba.neba-delivery-aem 3.11.0 zip {% endhighlight %}

    Or the Sling deployment bundle like so:

    {% highlight xml %} io.neba io.neba.neba-delivery-sling 3.11.0 {% endhighlight %}

    Finally, in order to install NEBA using the Content Package Maven Plugin you may do:

    {% highlight xml %} com.day.jcr.vault content-package-maven-plugin 0.0.24 io.neba io.neba.neba-delivery-aem 3.11.0 zip http://localhost:4502/crx/packmgr/service.jsp {% endhighlight %}
