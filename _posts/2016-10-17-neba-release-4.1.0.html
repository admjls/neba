Hightlights of this release

NEBA 4.1.0 ships with Spring 4.2.8

NEBA 4.1.0 updates Spring 4.2 to 4.2.8, the current GA (generally available) version (release notes).

Improved support for multiple version of the AEM Social Component Framework (SCF)

AEM ships with out of the box versions of the Social Component Framework that bring in a lot of third-party dependencies and package exports. Out of the box versions of SCF, for instance, include a partial export of the fasterxml Jackson libraries residing on the private classpath of one of the SCF bundles. This is incompatible with Spring, which (rightfully) relies on libraries being fully, not just partially, available. However, recent upgrades to the SCF fixed this issue by providing the jackson libraries as OSGi bundles.

This means NEBA needs to be able to run without these libraries, with a partial presence of these libraries and with the libraries being deployed as OSGi bundles. In 4.1.0, the corresponding imports in the Spring bundles shipped with NEBA were adjusted to meet these requirements, making NEBA function seamlessly regardless of the SCF deployment status.
