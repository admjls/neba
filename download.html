Installation in Adobe AEM™

The following ZIP file is a CRX package containing both NEBA and the runtime dependencies not provided by Sling, e.g. Spring and Gemini Blueprint.

Download the AEM™ package for NEBA {{site.nebaVersion}}

Installation in Apache Sling

The following JAR file is an OSGi bundle installing both NEBA and the runtime dependencies not provided by Sling, e.g. Spring and Gemini Blueprint.

Download the Sling delivery bundle for NEBA {{site.nebaVersion}}

Installing and activating this bundle installs NEBA and its runtime dependencies. Uninstalling the delivery bundle uninstalls NEBA as well.

Using Maven

NEBA is built using Maven. Core, API and delivery packages containing NEBA and it's runtime dependencies are available from maven central. For developing with NEBA, including the API in your project is all you need to do:

{% highlight xml %} io.neba io.neba.neba-api {{site.nebaVersion}} {% endhighlight %}

If you'd also like to browse or debug the core, add it to your project's dependencies like so:

{% highlight xml %} io.neba io.neba.neba-core {{site.nebaVersion}} {% endhighlight %}

The convenience CRX deployment package for AEM can be referenced like so:

{% highlight xml %} io.neba io.neba.neba-delivery-aem {{site.nebaVersion}} zip {% endhighlight %}

Or the Sling deployment bundle like so:

{% highlight xml %} io.neba io.neba.neba-delivery-sling {{site.nebaVersion}} {% endhighlight %}

Finally, in order to install NEBA using the Content Package Maven Plugin you may do:

{% highlight xml %} com.day.jcr.vault content-package-maven-plugin 0.0.24 io.neba io.neba.neba-delivery-aem {{site.nebaVersion}} zip http://localhost:4502/crx/packmgr/service.jsp {% endhighlight %}
